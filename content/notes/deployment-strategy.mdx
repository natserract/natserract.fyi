### DEVELOPMENT PHASE (2 weeks / 1 sprint)

1. Development Branch
2. Unit Test
3. Code
4. Linter
5. Self Testing (simulator)
6. MR Stage Branch
7. Code Review
8. Review & Approval
9. Build Pre-Alpha
10. Create Test Cases & Scenarios
11. Functional Testing
12. Build Alpha

### REGRESSION PHASE (~3 days)
1. Regression Testing
2. Fixing
3. Build Beta

### BETA PHASE (~2 days)
1. Beta / Internal Testing
2. Fixing
3. Build Release

### DEPLOYMENT PHASE (~1 week)
1. **Create Deployment Plan**
    1. Rollout 30% (Mon-Tue)
    2. Monitoring Crashlytics & ELK
    3. Fixing
    4. Build Patch
2. **Rollout 50% (Wed-Sun)**
    1. Monitoring Crashlytics & ELK
    2. Fixing
    3. Build Patch

3. **Rollout 100% (Mon)**
    1. Monitoring Crashlytics & ELK
    2. Fixing
    3. Build Patch
